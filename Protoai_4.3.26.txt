  🟡 Missing / Unimplemented Features

  1. Ingest workflow | Value: High | Difficulty: Medium

  The Rust backend exposes engine_ingest() (commands.rs:89-94) and the bridge has ingest() (engine_bridge.rs:176-178),
  but no UI calls it. The Tools tab has an "Ingest Code File" button (app.js:1061) that just opens the file picker
  instead of actually calling ingest. The server has IngestWorkflow.js but it's never registered in the workflow
  registry.

  2. DeepSearch, ImageGen workflows not registered in persistent IPC | Value: High | Difficulty: Easy-Medium

  These workflows exist in server/orchestration/workflows/ but are NOT registered in server.js or registerWorkflows.js.
  They currently work only via HTTP fallback routes in server-ipc.js. They should be registered in the main registry so
  the Tauri IPC path can invoke them directly.

  3. Browser tab in right pane | Value: Medium-High | Difficulty: Medium

  You requested this just now — iframe-based browser with URL bar, back/forward/reload, fallback-to-chat on
  X-Frame-Errors, state-preserving.

  4. Code Tools UI has no actual tools | Value: Medium | Difficulty: Easy-Medium

  The Tools tab has buttons for Spellcheck, Voice Chat, Version Info, Ingest Code, and Export Chat. But Ingest Code
  doesn't actually ingest — it just reopens the file picker. Voice Chat uses Web Speech API (browser only, won't work in
   Tauri WebView2 reliably).

  5. No persistent chat memory / context window | Value: High | Difficulty: Medium-Hard

  The memory system exists (/remember, MEMORY.md), but the app itself doesn't have in-chat memory, semantic search, or
  "remember this conversation" functionality. Chat history is flat JSON files. No ability to tag, bookmark, or search
  old messages.

  6. No real "Connectors" functionality | Value: Medium | Difficulty: Medium

  The "Connectors" chip exists (app.js:684-689) but it just sends a system-prompted chat. There are no actual configured
   connectors — no MCP servers, no API integrations, no LLM provider connections beyond the ones in ENGINES list. It's a
   fake feature — it just asks the LLM to "list connectors".

  7. No model output caching | Value: Medium | Difficulty: Medium

  No caching layer for repeated or similar prompts. Could save API costs and latency.

  8. No file preview/rendering | Value: Medium | Difficulty: Easy

  When files are attached, they show in the file list but can't be previewed. A file preview panel (text files, images,
  PDFs) in the right pane would provide value for the "files attached" workflow.

  9. No undo/redo in chat | Value: Low-Medium | Difficulty: Medium

  No ability to edit a sent message and re-send, or undo the last message. In Monaco editor there's undo, but the chat
  input is a plain textarea.

  10. Missing "Voice Chat" real integration | Value: Low-Medium | Difficulty: Medium

  Currently uses browser Web Speech API only. No actual TTS (text-to-speech) response from the AI — users only get text
  back. No voice recording, transcription, or playback.

  11. No streaming / incremental response rendering | Value: High | Difficulty: Hard

  Responses are displayed all-at-once after the LLM finishes. No streaming token-by-token display, no "thinking"
  indicator during generation — just the thinking blinker.

  12. Baseline CLI tools not migrated | Value: Low-Medium | Difficulty: Easy

  The baseline has baseline/cli/claude-select.js — a CLI tool for selecting Claude via environment. This didn't make it
  into the current build as a CLI or config feature.
